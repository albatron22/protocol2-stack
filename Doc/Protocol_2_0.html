Протокол взаимодействия уровней системы (верхний, нижний). Описание структуры.

Примечание 1. Данный протокол является развитием протокола взаимодействия верхнего и нижненго уровня подводного аппарата (вспоминай бессоные мартовские ночи...).

  • Версия: 2.0

  • Аппаратные настройки:

    • Интерфейс связи: UART
      • Порт UART CONTROL_PCB: USART2
      • Порт UART HL: /ttyTHS1
    • Параметры порта UART:
      • Скорсть: -

  • Общий формат пакетов

  • Система команд DL-канала

    • Задание линейного движения(главного вектора тяги) FORCE
    • Задание главного вектора момента MOM
    • Задание угловой скорости вращения ПА WR
    • Установка уровня яркости фонарей LAMP
    • Вращение камеры CAM
    • Сжатие схвата GRIP
    • Выбор режима MODE
    • Активация регулятора по положению REG
    • Перевод ПА в root-режим @ROOT

  • Up link канала - передача телеметрии TELEM

  • ROV Command Line Interface (rov-cli)

Связь двухсторонняя, асинхронная. Со стороны верхнего уровня (HL) передаются команды управления (Down Link канал - DL), со стороны нижнего уровяни (LL) CONTROL_PCB (Up Link канал - UL) показания датчиков (телеметрия) и ответы на запросы.

Общий формат пакетов

Пакеты сообщение имеют определённую структуру с символами начала, конца сообщения и полями . Данные передаются в преимущественно символьном виде (ASCII), то есть сообщения представляет собой строку. В пакете могут передаваться сообщения разного характера, который определяется в заголовке пакета - тип пакета.

Примечание. Версия протокола 2.0 основана на открытых протоколах Wialon IPS и NMEA.

Струкутура пакета:

$<TYPE>$<var_1,..,var_n><CR><LF>
Поле Описание Формат
$ Символ начала пакета (PRTCL2_START_SYMBOL). По умолчанию импользуется символ $, но может быть переопределён на дургой символ ASCII-символ
<TYPE> Тип пакета (PRTCL2_PKG_TYPE) ASCII-строка
$ Разделитель поля типа пакета и непосредественно содержания сообщения пакета (PRTCL2_PT_SYMBOL). Также может быть переропределён другим символом (например, в NMEA это запятая ,)
<var_1,..,var_n> Поле сообщения - непосредственно сами данные (сообщение), относящиеся к соответствующему заголовку. Внутри сообщения данные, если их несколько, разделяются знаком запятой ,. Данные в некоторых случаях могу отсутствовать, если заголовк считается командой. ASCII-строка и бинарные данные. Если бинарные, то разделитель отсутствует
<CR><LF> Символы конца сообщения (символы переноса каректи и строки \r\n) Спец. символы ASCII

Пример сообщения (без контрольной суммы CRC):

$FORCE$10,-20,30\r\n
$MOM$30,40,-50\r\n

Система команд DL-канала

В DL-канал HL отправляет команды LL. К этим командам относятся: задание линейного и вращательного движения, поворот камеры, включения фонаря и др.

Задание линейного движения(главного вектора тяги) FORCE

Линейное движение задаётся вектором силы тяги относительно системы координат подводного робота (ПР). Вектор имеет три компоненты - движение вперёд-назад (Fx), вправл-влево (Fy), вверх-вниз (Fz).

Описание пакета:

Поле Значение Формат Диапазон Ед. изм
Заголовок FORCE ASCII-строка
Данные Fx,Fy,Fz Символьный формат числа. Поддерживаются целочисленный формат и формат с плавающей точкой - ННН
Окончание команды \r\n Символ

Пример сообщения:

$FORCE$<Fx,Fy,Fz>\r\n
$FORCE$100.01,-200.2,300\r\n

Задание главного вектора момента MOM

Вращательное движение задаётся главным вектором момента вращения относительно системы координат подводного робота (ПР). Вектор имеет три компоненты - (Mx), (My), (Mz).

Описание пакета:

Поле Значение Формат Диапазон Ед. изм
Заголовок MOM ASCII-строка
Данные Mx,My,Mz Символьный формат числа. Поддерживаются целочисленный формат и формат с плавающей точкой - Н⋅мН\cdotмН⋅м
Окончание команды \r\n Символ

Пример сообщения:

$MOM$<Mx,My,Mz>\r\n
$MOM$10.1,20,30.33\r\n

Задание угловой скорости вращения ПА WR

Приведение ПА в требуемое угловое положение осуществляется вращением с заданной скоростью с джойстика. При отпускании стиков джойстика, ПА останавливается в текущем положении и стабилизируется в нём. Таким образом, угловой скоростью инкрементируется угловая ориентации ПА. В сообщении пакета передаются компоненты вектора угловой скорости ωr\mathbf{\omega}_rωr​ в разложении в базисе ПА: ωrx\omega_r^xωrx​, ωry\omega_r^yωry​, ωrz\omega_r^zωrz​.

Описание пакета:

Поле Значение Формат Диапазон Ед. изм
Заголовок WR ASCII-строка
Данные wx,wy,wz Символьный формат числа. Поддерживаются целочисленный формат и формат с плавающей точкой - рад/срад/срад/с
Окончание команды \r\n Символ

Пример сообщения:

$WR$<wx,wy,wz>\r\n
$WR$10.1,20,30.33\r\n

Перевод ПА в root-режим @ROOT

В этом режиме через последовательное соединение предоставляется команданый режим управлениями система ПА через интерфейс командной строки ROV CLI. Вместе с командой передаётся пароль root-режима.

Поле Значение Формат

Up link канала - передача телеметрии TELEM

Телеметрия передаётся одним пакетом. Формат пакета также определяется протоколом. Телеметрия, отправленная в UL-канале (от LL к HL) содержат обратную связь с датчиков и вычисленного угловй ориентации ПА. К датчикам относятся: IMU сенсор, датчики внешнего и внутреннего давления, датчики внешенй и внутренней температуры, уровень заряда АКБ. Вычисление углового положения происходит на стороне CONTROL_PCB. Значение углового положения определяется кватернионом собственного положения.

Пакет телеметрии UL-канала представляет собой строку общего формата, в которой содержатся все возможные передаваемые данные с датчиков и оценка угловой ориентации (кватернион собственного положения - по отношению к Земле). Порядок данных строго определён. Данные следуют за заголовком пакета (типа пакета). Данные имеют числовой тип (целочисленный и вещественный).

Описание пакета:

Поле Значение Формат
Заголовок TELEM ASCII-строка
Данные Q_w,Q_x,Q_y,Q_z,wx,wy,wz,ax,ay,az,hx,hy,hz,t_out,depth,v,mode,reg Символьный формат числа. Поддерживаются целочисленный формат и формат с плавающей точкой
Окончание команды \r\n Символ

Содержание пакета телеметрии

Поле Значение Формат Диапазон Ед. изм.
Q_w Действительная часть кватерниона w Вещественное в символьном виде (float) - -

Q_y Мнимая часть кватерниона y Вещественное в символьном виде (float) - -
Q_z Мнимая часть кватерниона z Вещественное в символьном виде (float) - -

ax, ay, az Показания акселерометра - проекция ускорения движенгия на связанные оси ПР Вещественное в символьном виде (float) - g

ROV Command Line Interface (rov-cli)

После перевода ПА в root-режим становится доступным интерфейс командной строки для управления системами ПА. Команды обязательно должны завершаться символами CRLF \r\n

Список команд

  1. exit - выход из rov-cli
  2. motor <ch> <value> - управление движителем (<ch> - канал, к которому подлкючен движитель 1...8, <value> - тяга в граммах -1800...1800)
  3. lamp <ch> <value> - управление фонарём (<ch> - канал, к которому подлкючен фонарь 1...3, <value> - уровень яркости 0...100 %)
  4. grip <value> - управление схватом (<value> - скорость сжатия -100...100, 0 - останов!)
  5. servo <ch> <value> - управление сервоприводами подвеса (<ch> - канал сервопривода 1, 2, <value> - угол поворота 0...180 градусов)
  6. control coef get - считывание текущих значений коэффициентов регулятора положения
  7. control coef set <kwp> <kwi> - задание коэффициентов регулятора (<kwp> - коэф. ПИ-регулятора конутра угловой скорости (П), <kwi> - коэф. ПИ-регулятора конутра угловой скорости (И))
  8. get pos - считывание ткущей позиции в углах Эйлера roll (крен), pitch (тангаж), yaw (курс)
